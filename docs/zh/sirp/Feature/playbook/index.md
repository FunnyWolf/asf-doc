# Playbook

与 Alert/Case/Artifact 相关联的自动化剧本记录.

## Kanban / View

![img.png](img.png)

## Detail

- Status

Playbook 当前状态,分为 `Pendding` `Running` `Success` `Failed` 四种状态.

- Type

`Case` `Alert` `Artifact` 三种类型.

- Name

执行的 Playbook 名称.

- Worksheet

关联的 Worksheet.

- Job ID

ASP 后台生成的唯一 Job 编号.

- Row ID

关联的 Worksheet rowId

- Remark

任务执行的备注信息.





- 打开详情页,点击左上角的 `Playbook` 按钮.


- 选择需要执行的 Playbook,点击 `确认` 按钮.


- 任务初始状态为 `Pending` ,等待调度执行.


- 任务执行过程中,状态为 `Running`.


- 任务执行完成后,状态为 `Success` 或 `Failed`,点击任务记录可查看执行详情.
